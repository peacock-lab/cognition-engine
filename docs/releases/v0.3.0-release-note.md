# v0.3.0 Release Note

## Version Positioning

`v0.3.0` is the Cognition Engine ADK foundation alignment and local real-model workflow stabilization release.

This release does not claim a complete productized platform. It does not claim public provider selection, completed Eval, complete Observability, or a formal configuration center.

## Highlights

This release stabilizes the following capabilities:

1. ADK-backed workflow main path;
2. install-mode `CE_DATA_DIR` workflow execution;
3. `CE_INSIGHTS_DIR` as a fine-grained insight directory override;
4. output and metadata generation path;
5. ADK session / invocation / events / artifacts mapping;
6. no legacy fallback on the verified workflow path;
7. explicit real-provider entry through `CE_MODEL_PROVIDER=adk_litellm_ollama`;
8. minimum ADK b1 framework metadata entry.

## Minimal Usage

Source-tree workflow:

```bash
CE_DATA_DIR="$PWD/data" python -m cognition_engine.cli workflow --insight insight-adk-runner-centrality --json
```

Real-provider workflow is an advanced local-environment path and requires local Ollama and the matching model:

```bash
CE_DATA_DIR="$PWD/data" \
CE_MODEL_PROVIDER=adk_litellm_ollama \
CE_MODEL_TIMEOUT_SECONDS=180 \
OLLAMA_API_BASE=http://127.0.0.1:11434 \
python -m cognition_engine.cli workflow --insight insight-adk-runner-centrality --json
```

The default provider remains `mock`. The `--model-provider` CLI option is not part of the public CLI surface in this release.

## Data Boundary

The current ADK dependency line is:

```text
google-adk>=2.0.0b1,<2.1
```

`data/frameworks/adk-2.0.0b1/metadata.json` is the minimum b1 framework metadata entry.

The `adk-2.0.0a3` insight samples remain as historical smoke / fixture / regression assets. They are not renamed or copied as b1 insight samples.

## Deferred

This release defers:

1. public provider selection;
2. public `--model-provider` CLI option;
3. Eval;
4. complete Observability;
5. Runner / observation / context system-level governance;
6. b1 insight sample system;
7. formal configuration center.

## Next Direction

`v0.4.0` is expected to focus on ADK runtime governance across three lines:

1. Runner;
2. observation;
3. context.

Artifacts and events remain a minimum evidence path in `v0.3.0`; systematic treatment moves to the `v0.4.0` observation line.
