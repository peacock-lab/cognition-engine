# v0.3.1 Release Note

## 1. Version positioning

`v0.3.1` is a small post-release fix after `v0.3.0`.

This version focuses on the local-model dependency path.

It does not introduce a new product line, does not change the default provider, and does not expose the `--model-provider` CLI option.

---

## 2. What changed

`v0.3.1` changes the LiteLLM dependency strategy:

1. LiteLLM is promoted from the optional `llm` extra to the base dependencies;
2. LiteLLM is locked to `litellm==1.82.6`;
3. regular installation now includes the dependency required by the local-model path;
4. the default provider remains `mock`;
5. a real provider still requires explicit environment-variable enablement;
6. `--model-provider` remains outside the public CLI surface.

---

## 3. Why this change was made

After the `v0.3.0` release, user-perspective testing showed that:

```bash
python -m pip install cognition-engine==0.3.0
```

could run the default mock workflow path, but the explicit local-model provider path required LiteLLM.

In `v0.3.0`, LiteLLM was placed under the optional `llm` extra, so a regular installation did not install it.

For the target user path, this was too easy to miss.

Starting from `v0.3.1`, regular installation includes LiteLLM directly.

---

## 4. LiteLLM version

The selected version is:

```text
litellm==1.82.6
```

Reason:

1. `litellm==1.82.6` installs successfully in the project Python 3.14 environment;
2. local-model workflow testing passed with `litellm==1.82.6`;
3. `litellm==1.83.7` was not adopted because it pins `pydantic==2.12.5`, which conflicts with the current `pydantic>=2.13.0` baseline;
4. `litellm` 1.83.10 and later were not adopted in this release because their current PyPI metadata declares `Requires-Python <3.14`, which blocks installation in the current Python 3.14 environment;
5. future LiteLLM upgrades will be reconsidered after LiteLLM restores compatibility with Python 3.14 and Pydantic 2.13+.

---

## 5. Installation

Install the current version:

```bash
python -m pip install cognition-engine==0.3.1
```

Confirm the CLI:

```bash
ce --help
python -m cognition_engine.cli --help
```

---

## 6. Default workflow path

The default provider remains:

```text
mock
```

Recommended minimal workflow:

```bash
CE_DATA_DIR="$PWD/data" ce workflow --insight insight-adk-runner-centrality --json
```

---

## 7. Local-model path

The local-model path no longer requires installing an extra package group.

Regular installation already includes LiteLLM.

A real provider can be explicitly enabled with:

```bash
CE_MODEL_PROVIDER=adk_litellm_ollama \
CE_DATA_DIR="$PWD/data" \
CE_INSIGHTS_DIR="$PWD/data/insights" \
CE_MODEL_TIMEOUT_SECONDS=180 \
OLLAMA_API_BASE=http://127.0.0.1:11434 \
ce workflow --insight insight-adk-runner-centrality --json
```

The local-model path still requires:

1. a running local Ollama service;
2. an available local model;
3. explicit `CE_MODEL_PROVIDER=adk_litellm_ollama`.

---

## 8. Not changed

`v0.3.1` does not change:

1. the default provider;
2. the public CLI provider surface;
3. `--model-provider` visibility;
4. the ADK-backed workflow boundary;
5. the public release asset strategy;
6. the data asset boundary.

---

## 9. Summary

`v0.3.1` is a small dependency and documentation fix: LiteLLM is now part of the base installation, locked to `1.82.6`, so users can explicitly enable the local-model provider path without installing an optional extra.
