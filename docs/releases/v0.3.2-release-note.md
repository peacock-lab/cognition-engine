# v0.3.2 Release Note

## 1. Version positioning

`v0.3.2` is a small dependency and virtual-environment governance release after `v0.3.1`.

This version focuses on uv-based dependency locking, virtual-environment synchronization rules, and clean pre-release environment verification.

It does not introduce a new product capability, does not change the default provider, and does not change the local-model dependency strategy established in `v0.3.1`.

---

## 2. What changed

`v0.3.2` introduces the following environment-governance changes:

1. added `uv.lock` to both the internal repository and the public repository;
2. upgraded the user-level `uv` tool to `0.11.8`;
3. clarified that `uv` is a user-level global tool at `/Users/peacock/.local/bin/uv`;
4. clarified that `uv` is not installed into each repository's `.venv`;
5. preserved separate `.venv` directories for the internal and public repositories;
6. added `twine>=6.2.0` to the `release` extra;
7. documented scenario-based `uv sync` commands;
8. verified `uv sync --all-extras` in both repositories;
9. documented clean pre-release environment verification rules;
10. kept user-perspective installation tests based on regular `pip install`.

---


Current version:

## 5. uv sync rules

`uv sync` should be used by scenario:

```bash
# Runtime-only environment
uv sync

# Test environment
uv sync --extra test

# Release environment
uv sync --extra release

# Full development / test / release environment
uv sync --all-extras
```

Default `uv sync` only syncs base dependencies and does not install optional extras.

Therefore, release preparation should not blindly rely on default `uv sync`.

## 7. Clean pre-release verification

Pre-release validation now has two separate tracks:

```text
development repository validation: uv-based
user-perspective validation: pip-based
```

Development / release validation may use:

```bash
uv sync --all-extras
```

User-perspective validation must still use a clean environment and regular installation:

```bash
python -m pip install cognition-engine==0.3.2
```

The two tracks are not interchangeable.
