# v0.2.0 Release Note

## Release type

v0.2.0 is a staged milestone release for Cognition Engine.

This release focuses on the first ADK-backed cognitive workflow milestone. It is not a full external major release and does not expose internal real-provider configuration.

## Highlights

1. Added the staged ce workflow capability.
2. Connected product brief, decision pack, and model enhancement into one insight-to-decision workflow.
3. Added workflow-level Markdown summary output.
4. Added workflow-level metadata.
5. Added the ADK-backed workflow adapter as the minimum workflow backbone.
6. Added workflow execution, session/context/invocation mapping fields.
7. Added event summary and artifact reference fields in workflow results.
8. Kept the public CLI surface narrow and stable.

## Public CLI

The public workflow entry remains:

    ce workflow --insight <insight_id>
    ce workflow --insight <insight_id> --json

This release does not expose real-provider CLI parameters.

## Output locations

The main output locations are:

    outputs/product-briefs/
    outputs/decision-packs/
    outputs/model-enhancements/
    outputs/workflows/
    outputs/.metadata/

outputs/workflows/ stores workflow-level Markdown summary files.

outputs/.metadata/ stores metadata records, including workflow-level metadata.

## ADK-backed workflow boundary

v0.2.0 introduces the minimum ADK-backed workflow bridge.

This means the workflow path is no longer only a local Python sequential orchestration path. It now carries ADK-backed execution semantics and exposes workflow result fields for execution, session/context/invocation mapping, event summary, and artifact references.

This does not mean the project has completed every ADK capability.

## Not included in this release

This release does not include:

1. Public real-provider usage.
2. Ollama or Gemma setup instructions.
3. --model-provider or --model workflow CLI parameters.
4. Batch workflow.
5. Workflow dashboard.
6. Full ADK Eval integration.
7. Full Artifact store replacement.
8. Long-term session persistence.
9. Multi-agent collaboration.

## Validation scope

The private release validation covered:

1. Product brief loop.
2. Decision pack loop.
3. Model enhancement loop.
4. Workflow loop.
5. CLI workflow dispatch.
6. ADK workflow adapter tests.
7. Dependency check.
8. Output residue check.

The public repository keeps the user-facing documentation, core code assets, and relevant tests needed for this staged milestone.
